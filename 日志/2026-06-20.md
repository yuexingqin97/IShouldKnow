## 今天学的

### 原型链（从"会用"到"通了"）

- `prototype` 是函数的属性（模板），`__proto__` 是对象的长处（链指针）
- `class extends` 搭两座桥：实例桥 + 静态桥
- **函数也是对象**——这是理解静态桥的关键。`Dog.__proto__ === Animal`，跟 `d.__proto__ === Dog.prototype` 是同一套机制
- 字段的读/写不对称：读走原型链，写永远塞实例（shadowing）；引用类型是共享坑——所以 Vue data 必须是函数
- `super()` = `Parent.call(this)`：父类字段靠这行跑到子实例上
- `this` 四条铁律：默认/隐式/显式/箭头函数。传回调用 `bind` 锁 this
- `#` 私有字段：TS private 是建议（编译完没了），JS `#` 是法律（V8 强制执行）
- JS `get`/`set` 是 ES5 原生，C# backing field 自动生、JS 必须手写 `_name`
- 原型链污染 + 防御：`Object.create(null)` / `hasOwnProperty` / `Map`
- 内置类型全家谱：Array/Function/Error 各走各的 `Xxx.prototype` → 最终汇入 `Object.prototype`

### V8 底层

- Hidden Class + Inline Cache：属性按相同顺序添加 → 共享 Hidden Class，JIT 把原型链查找缓存为直接偏移
- 单态（1 种 HC）→ 多态（2-4 种）→ 巨态（>4 种退化回哈希）——保持类结构稳定 = 帮 V8 作弊
- 普通 Array：PACKED_SMI → PACKED_DOUBLE → HOLEY，降级不可逆。空洞用 `splice` 别用 `delete`
- TypedArray：绝对连续内存，跟 C++ `int buf[N]` 对等。ArrayBuffer 多视图共享同一块内存

### TS 特性

- 数字枚举有反向映射（`Color[0]` → `'Red'`），字符串没有
- 枚举编译产物是 IIFE——为了"赋值当表达式 + 反向映射 + 声明合并"三个需求
- `const enum` 运行时不存在，`as const` 是干净替代
- TS namespace 是运行时对象（IIFE 往同一个对象加属性），跟 C# 的编译期符号前缀完全不同

### 模块系统

- ESM 活绑定 vs CJS 值拷贝
- 循环依赖炸的本质：模块加载顺序改变 → 某个 class 在用到时还是 `undefined`
- 三种解：抽接口（最优）→ Registry/DI（游戏最实用）→ 动态 import（止血）
- barrel export：TS 替代 C# namespace 的方式（`import * as F from './f'` → `F.A` / `F.B`）

### Puerts 通信全路径

- C++ → TS 六种方案：DYNAMIC_DELEGATE / toDelegate(3 种) / std::function / BlueprintFunctionLibrary / Override / Mixin
- `std::function` 导出不需要 UFUNCTION，靠 `DefineClass<XXX>().Method("name", MakeFunction(&XXX::Method)).Register()`
- `FJsObject::Get/Set` 走 V8 属性表查找，比 `std::function` 回调少一层函数调用
- TS → C++ 三条路：V8FastCall（直接 C 函数指针）/ FastCall（手拼 FFrame → 原生 Func）/ SlowCall（ProcessEvent）
- 反射数据解析一次缓存为 `PropertyTranslator` 数组，后续调用复用
- 结构体过边界必拷贝（`PassByPointer=false` → new + CopySingleValue），UObject 不拷贝
- FString/FName/FText：FName 大小写不敏感，传引用走 ArrayBuffer 零拷贝；FText 默认丢本地化

### Override 继承链（JS 原型 + C++ 类的完整链路）

```
class extends (JS 桥) → makeUClass (提取方法) → UTypeScriptGeneratedClass (C++ 子类)
→ JsConstruct (SetPrototype + 调 TS 构造) → ObjectInitialize (换函数指针)
→ 运行时 execCallJS → InvokeTsMethod → ObjectMap 查 JS 对象 → 沿原型链找方法
```

### Mixin

- Mixin 不走原型链：`defineProperty` 拷函数到 `jsCls.prototype`，不设 `__proto__`
- C++ 侧替换 UFUNCTION 的 `NativeFunc` 为 `execCallMixin`
- 每次调用比 Override 多一步字节码解析（从 Script 数组里读 magic marker → 反序列化指针）
- 性能不如 Override，但优势是不改类体系

### `blueprint.tojs`

- 蓝图类运行时 `Field.Load` 后只是 C++ 指针，没有 JS FunctionTemplate 绑定
- `tojs` → `UETypeToJsClass` → `GetTemplateInfoOfType`：遍历 UFUNCTION、创建 FFunctionTranslator、生成 V8 FunctionTemplate + 绑定方法
- C++ 原生类不需要——Puerts 启动时已自动绑定
