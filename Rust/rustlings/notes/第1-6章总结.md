## 1. 变量与常量

### `let` — 声明的基础

```rust
let x = 5;         // 类型推导为 i32
let x: i32 = 10;   // 显式类型注解
```

记住：Rust **没有** C/JS 的裸赋值声明，必须写 `let`。

### `mut` — 可变性

```rust
let mut x = 3;
x = 5;  // ✅ 可以改
```

默认不可变（Rust 的保守设定），可以改才标 `mut`。C++ 正好反过来：`const` 是主动声明不变的标记。

### Shadowing（变量遮蔽）

```rust
let number = "T-H-R-E-E";    // &str
let number = 3;              // i32，遮蔽了上一行
```

同一个名字可以重新 `let` 绑定，类型可以不同。这和 C++ 的 `int x = 1; { string x = "hi"; }` 同名冲突不同——Rust 的 shadowing 是同作用域内的**新绑定**，旧的值被"遮住"但还在内存里（如果没被移动）。比 `mut` 更灵活——可以改类型。

> C++ 对比：C++ 不允许同作用域内用同名变量。JS 的 `var` 允许但 `let` 不允许。Rust 通过显式 `let` 区分了"重新绑定"和"赋值"。

### `const` — 编译期常量

```rust
const NUMBER: i32 = 3;    // 必须写类型注解
```

和 `let` 的区别：`const` 是**编译期**的，`let` 是**运行时**的；`const` 不能 `mut`。

## 2. 函数

### 定义和调用

```rust
fn call_me() { ... }           // 无参无返
fn call_me(num: i32) { ... }   // 有参数
fn sale_price(price: i64) -> i64 { ... }  // 有返回值
```

参数**必须写类型**，没有 C++ 的默认参数。

### 两种返回方式

```rust
// 表达式返回：不加分号
fn square(num: i32) -> i32 {
    num * num      // ← 没有 ;  → 这是返回值
}

// 显式返回：return 关键字
fn square(num: i32) -> i32 {
    return num * num;   // ← 可以加分号，因为有 return
}
```

如果最后一行加了 `;` 但没有 `return`，返回值变成 `()`（unit），和签名里的 `-> i32` 冲突。

> C++ 对比：C++ 的 `return` 是唯一的返回方式。Rust 多了一种"表达式返回"——最后一个表达式不加分号就是返回值。习惯了 JS 的话想想箭头函数 `(x) => x * x` 的效果。

## 3. if 是表达式

**Rust 里 `if` 有值，不是语句。**

```rust
let bigger = if a > b { a } else { b };  // 整个 if 是一个 i32 值
```

### 核心规则：每个分支的类型必须相同

```rust
// ❌ 不行：i32 和 &str 不是同类型
let x = if cond { 1 } else { "two" };

// ✅ 可以：全是 i32
let x = if cond { 1 } else { 2 };
```

这就是你 `if2.rs` 碰到的报错——`1` 和 `"not a crab"` 类型不一致。

### 支持 else if 链

```rust
let habitat = if animal == "crab" { 1 }
          else if animal == "gopher" { 2 }
          else if animal == "snake" { 3 }
          else { 0 };
```

## 4. 基本类型

### bool

```rust
let is_morning = true;
let is_evening = !is_morning;   // 取反
```

### char — 4 字节 Unicode

```rust
let c = 'A';         // ASCII
let e = '😊';        // emoji（注意单引号，不是双引号）
```

Rust 的 `char` 是 4 字节（Unicode scalar value），不是 C 的 1 字节。`"string"`（双引号）→ `&str`，`'c'`（单引号）→ `char`。

### 数组 `[T; N]` — 固定长度，编译期确定

```rust
let a = [1, 2, 3, 4, 5];    // [i32; 5]
let zeros = [0; 100];        // [i32; 100]，100 个 0（repeat 表达式）
a.len()                       // 5
```

### 切片 `&[T]` — 数组的"视图"，不拥有数据

```rust
let nice_slice = &a[1..4];   // [2, 3, 4]，索引 1~3（不含 4）
```

切片是比较内容的（`PartialEq<[T]>`），不是比较地址——你之前担心的 `==` 问题，Rust 的答案是逐元素比。

### 元组 `(T1, T2, ...)`

```rust
let cat = ("Furry McFurson", 3.5);  // (&str, f64)
let (name, age) = cat;              // 解构
let second = numbers.1;             // 索引访问（从 0 开始）
```

## 5. Vec — 动态数组

> Vec 对应 C++ 的 `std::vector`，可以增长。

```rust
let v = vec![10, 20, 30, 40];   // vec! 宏创建
let mut v2 = Vec::new();         // 空 vec
v2.push(42);                     // 末尾添加
```

### 迭代方式

```rust
// for 循环
for element in &slice {
    output.push(element * 2);   // element 自动解引用，可以直接 * 2
}

// 迭代器链
slice.iter().map(|element| *element * 2).collect()
//                    ↑ 这里 * 是必须的：iter() 给的是 &i32 引用
```

> `iter()` 返回 `&T` 的引用，所以 map 闭包拿到的是 `&i32`，需要 `*` 解引用才能做乘法。而 `for el in &slice` 里 `el` 也会被解引用（编译器帮你）。

## 6. 所有权 — move_semantics 的核心

**这是 Rust 和 C++/JS 最大的分水岭。** 建议反复看。

### 规则一：赋值 = 移动所有权（默认）

```rust
let vec0 = vec![1, 2, 3];
let vec1 = vec0;         // vec0 把所有权「移动」给了 vec1
// vec0 不能再用了！     // vec0 已失效，编译器报错
```

C++ 里同样代码是**拷贝**（或浅拷贝导致 double free），Rust 默认是**移动**——零开销，旧变量失效。

> C++ 对照：`auto v2 = std::move(v1);` 之后 `v1` 是"valid but unspecified"——编译器不强制你忘掉它，但用了是 bug。Rust 把这条规则**编译期强制执行**——`v1` 移动后，编译就报错。

### 规则二：`mut` 参数 = 拿所有权 + 可改

```rust
fn fill_vec(mut vec: Vec<i32>) -> Vec<i32> {  // ← mut 可以不加新变量
    vec.push(88);
    vec                  // 返回所有权给调用方
}
```

这是 `move_semantics3` 的解法——最干净，零 clone。

### 规则三：`&T` 借用 = 借用读权，不移动

```rust
fn fill_vec(vec: &[i32]) -> Vec<i32> {
    let mut vec = vec.to_vec();   // 基于引用创建新 Vec
    vec.push(88);
    vec
}

let vec0 = vec![1,2,3];
let vec1 = fill_vec(&vec0);      // 传引用——vec0 还能用
assert_eq!(vec0, [1,2,3]);       // ✅ vec0 还在
```

`move_semantics2` 的解法。`to_vec()` 在堆上分配了一个新的 Vec，不是浅拷贝。

### 规则四：`&mut T` 可变借用 = 独占写权

```rust
let mut x = Vec::new();
let y = &mut x;       // 可变借用
y.push(42);           // y 用完了——NLL 规则，借用结束
let z = &mut x;       // 可以再次可变借用，因为 y 已经失效
z.push(13);
```

`move_semantics4` 的核心：**同时只能有一个可变借用**。但因为 NLL（Non-Lexical Lifetimes），编译器追踪最后一次使用——`y` 在 `y.push(42)` 之后再没出现，所以 `z` 可以立即借用。

> C++ 对照：这就是 C++ 的"不要同时持有两个可写引用"变成编译期规则。C++ 里你写出这种代码，运行时可能没 bug 也可能有——编译器不管。Rust 直接不让你编译。

### 规则五：给引用不等于给所有权

```rust
fn get_char(data: &String) -> char {  // 只借用，不占据所有权
    data.chars().last().unwrap()
}

fn string_uppercase(mut data: String) {  // 拿走所有权
    data = data.to_uppercase();          // to_uppercase() 返回新 String
    println!("{data}");
}

let data = "Rust is great!".to_string();
get_char(&data);          // 只借——data 还在
string_uppercase(data);   // 移走——data 失效
```

`move_semantics5` 的做法。调用方传 `&` 还是传所有权，完全取决于函数要不要"吃完"这个值。

### 解法对照：同一道题三种改法

以 `move_semantics1-3` 为例，同一道逻辑题，Rust 有三种编译通过的方式：

| 方式 | 代码 | 什么时候用 |
|------|------|-----------|
| clone | `let mut vec = vec.clone();` | 调用方之后还要用原 vec（本题不是） |
| 借用 | `fn fill_vec(vec: &[i32])` | 只读不写原 vec |
| **拿走 + mut 参数** | `fn fill_vec(mut vec: Vec<i32>)` | 调用方不需要原 vec——最干净 |

> `move_semantics1` 你的 clone 写法**编译正确、逻辑正确**，但不是最优解——因为调用方之后 **没用** `vec0`，白复制了一次。`move_semantics3` 的 `mut vec` 是这道题的"规范答案"：拿走所有权、就地修改、返回。但这属于**效率优化**，不是错误——Rust 允许你 clone，只是提醒你这样做了。
