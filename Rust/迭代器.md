# 迭代器

> 谈了 `for` 怎么脱糖、适配器链为什么零成本、生命周期怎么传、`collect` 七十二变的原理。

## `for` 脱糖和 `IntoIterator`

```rust
for item in xxx { }
// ↓ 等价于
let mut iter = <xxx as IntoIterator>::into_iter(xxx);
while let Some(item) = iter.next() { }
```

**`for` 要的是 `IntoIterator`，不是 `Iterator`。** 但所有 `Iterator` 自动实现了 `IntoIterator`（blanket impl），所以迭代器也能直接放进 `for`。

### 三种实现（以 `Vec` 为例）

```rust
impl IntoIterator for Vec<T>        { type Item = T;     }  // for item in v
impl IntoIterator for &Vec<T>       { type Item = &T;    }  // for item in &v
impl IntoIterator for &mut Vec<T>   { type Item = &mut T;}  // for item in &mut v
```

三份 impl 换的不是方法名，是 `Self`。`self` 是引用，`Item` 自然就是引用——类型系统推导，不靠运行时。

## `Iterator`：只要求 `next`

```rust
pub trait Iterator {
    type Item;
    fn next(&mut self) -> Option<Self::Item>;  // ← 唯一必须实现的
    // map/filter/collect/fold/sum... 都有基于 next 的默认实现
}
```

`&mut self` 是因为迭代器内部游标要前进，`Option` 表达"有没有下一个"。

和 C++ 的对应：

```cpp
++it;       // → next() 里的指针前进
*it;        // → Some(val)
it == end;  // → None
```

Rust 把"移动/取值/判断结束"合并到一个调用。

## 适配器链：惰性 + 零成本

```rust
v.iter()
    .filter(|x| *x % 2 == 0)
    .map(|x| x * 10)
    .take(2)
    .collect::<Vec<_>>();
```

- **每步产生一个栈上 struct**（几字节），不做堆分配，不拷贝数据
- **惰性求值**：`filter`/`map`/`take` 只是搭调用链，`collect` 才触发真正遍历
- **编译器内联**：泛型单态化后，整条链被内联成单个紧凑循环，运行时没有适配器开销

### 和 TS/JS 关键区别

```ts
// JS: 每步都产生一个新数组
arr.filter(f)   // 分配新数组
   .map(g)      // 又分配新数组
   .slice(0,2); // 又分配
```

JS 的方法签名叫 `Array`，没法返回"还没算完的东西"。Rust 的类型系统允许返回 `Filter<Iter, F>` 这样的惰性适配器——这是静态类型带来的福利。

### 常用适配器速查

| 适配器 | 用途 |
|---|---|
| `filter` | 按条件过滤 |
| `map` | 逐元素变换 |
| `take` / `skip` | 取前 N 个 / 跳过前 N 个 |
| `enumerate` | 加索引 `(i, item)` |
| `zip` | 两个迭代器并行，短的那个到头就停 |
| `flat_map` | 每个元素展开成多个，整体展平 |
| `chain` | 拼接两个迭代器 |
| `peekable` | 看下一个不消费 |
| `rev` | 反转（背后是 `DoubleEndedIterator`） |

## 生命周期：借用迭代 vs 所有权迭代

```rust
let v = vec![1, 2, 3];

let iter = v.iter();       // Iter<'_, i32> —— 有生命周期参数，借用了 v
let iter = v.into_iter();  // IntoIter<i32> —— 没有生命周期参数，拥有数据
```

- **`Iter`**：不能比原容器活得久，适配器链透传生命周期
- **`IntoIter`**：拥有数据，可以带出函数，没有生命周期约束
- 除此之外迭代器没有引入新的生命周期规则——就是借用检查器照常工作

## `FromIterator`：collect 七十二变的原理

```rust
pub trait FromIterator<A> {
    fn from_iter<T: IntoIterator<Item = A>>(iter: T) -> Self;
}
```

`collect()` 就是反过来调 `FromIterator::from_iter()`。

### 不同目标类型的实现策略

| 目标类型 | 策略 |
|---|---|
| `Vec<T>` | 逐个 push |
| `HashMap<K, V>` | 遍历 `(K, V)` 对，逐个 insert |
| `String` | 逐个 push char |
| `Result<Vec<A>, E>` | 逐个处理，遇到第一个 `Err` 就**短路返回** |
| `Option<Vec<A>>` | 遇到第一个 `None` 短路 |

`Result` 的短路是最妙的：`collect` 直接收集成 `Result<Vec<_>, _>`，不用手写 `for` + `match` + `return`。

---

## 三层架构

```
Iterator::next()        ← 只要求实现这个
      ↓
 各种适配器             ← map/filter/take... 惰性，栈上 struct
      ↓
   collect()            ← 消费整条链
      ↓
FromIterator::from_iter()  ← 决定收成什么
      ↓
 Vec / HashMap / Result<Vec<_>, E> / String / ...
```

迭代器体系就这三层。没有运行时魔法，全是编译期泛型 + trait 的组合。
