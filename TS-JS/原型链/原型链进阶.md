# 原型链进阶：污染漏洞 + 内置类型家谱

> 读完原型链基础之后再翻这篇——两篇是姐妹篇。

## 原型链污染 —— JS 特有的安全漏洞

一句话：**改了 `Object.prototype`，所有对象都中招。**

```js
Object.prototype.isAdmin = true;  // 某段代码干了这件事

const user = { name: '小明' };
user.isAdmin;          // true —— 从原型链读到
user.isAdmin = false;  // 写操作塞在实例上，遮蔽了原型

// for...in 也会遍历原型上的属性
for (const key in user) {
  console.log(key);  // 'name', 'isAdmin'
}
```

### 怎么被利用：`merge` 漏洞

```js
function merge(target, source) {
  for (const key in source) {
    if (typeof source[key] === 'object') {
      target[key] = merge(target[key] || {}, source[key]);
    } else {
      target[key] = source[key];  // ← key 是 '__proto__' 时会改原型链
    }
  }
  return target;
}

const malicious = JSON.parse('{"__proto__": {"isAdmin": true}}');
merge({}, malicious);

const x = {};
x.isAdmin;  // true —— 原型被污染了
```

> `__proto__` 是字符串 key，但 `target['__proto__'] = value` 不是设属性——它直接替换 `target` 的原型链。这是 JS 语言历史上的一个设计失误（现在应该用 `Object.getPrototypeOf` / `Object.setPrototypeOf`）。

### 防御

```js
// 1. Object.create(null) —— 纯字典，没有原型链
const safe = Object.create(null);
safe.__proto__;    // undefined
safe.toString;     // undefined

// 2. for...in 加 hasOwnProperty 过滤
for (const key in obj) {
  if (!Object.prototype.hasOwnProperty.call(obj, key)) continue;
}

// 3. 用 Map 替代对象做字典
const m = new Map();
m.set('key', 'value');  // Map 不受 Object.prototype 污染影响
```

## 内置类型的原型链家谱

你天天用 `Array`、`Function`、`Error`——它们的原型链各自长什么样？`arr.map()` 实际在哪找到的？

### 全网

```
                                    null
                                      ↑
                              Object.prototype    ← hasOwnProperty, toString
                              ↑             ↑
                      Array.prototype   Function.prototype  ← call, apply, bind
                      (map, filter)     ↑          ↑
                          ↑           Dog        Animal
                      [1, 2, 3]        ↑          ↑
                                   Dog.prototype  Animal.prototype
                                          ↑
                                      d (实例)
```

### Array 实例

```bash
node -e "
const arr = [1, 2, 3];
console.log('arr.__proto__ === Array.prototype:', arr.__proto__ === Array.prototype);
console.log('arr.map === Array.prototype.map:', arr.map === Array.prototype.map);
// Array 重写了 toString
console.log('arr.toString 来自:', arr.toString === Array.prototype.toString);
"
```

```
arr.__proto__ → Array.prototype    ← map/filter/reduce/splice 在这里
       ↓
   Object.prototype                ← toString（但 Array.prototype 重写了）
       ↓
   null
```

### Function 实例

```bash
node -e "
function Dog() {}
console.log('Dog.__proto__ === Function.prototype:', Dog.__proto__ === Function.prototype);
console.log('Dog.call === Function.prototype.call:', Dog.call === Function.prototype.call);
// Dog.prototype 是普通对象，它的 __proto__ 指向 Object.prototype
console.log('Dog.prototype.__proto__:', Dog.prototype.__proto__ === Object.prototype);
"
```

```
Dog (函数)
  ├─ __proto__ → Function.prototype  ← call/apply/bind 在这里
  │                  ↓
  │              Object.prototype
  │                  ↓
  │                null
  │
  └─ prototype → Dog.prototype       ← new Dog() 产生的实例走这条
                      ↓
                  Object.prototype
```

> 关键区分：`Dog.__proto__`（函数找静态方法的链）≠ `Dog.prototype.__proto__`（实例找方法的链）。两条不一样的链，同一个 `Object.prototype` 收尾。

### Error 实例

```bash
node -e "
const e = new Error('oops');
// Error.prototype → Object.prototype → null
console.log('e.__proto__ === Error.prototype:', e.__proto__ === Error.prototype);
// Error 可以被子类化（ES6 class extends 解决了）
class MyError extends Error {
  constructor(msg, code) { super(msg); this.code = code; }
}
const me = new MyError('custom', 404);
console.log('me instanceof Error:', me instanceof Error);  // true
console.log('me.code:', me.code);
console.log('me.stack:', me.stack ? 'has stack' : 'no stack');
"
```

```
e.__proto__ → Error.prototype     ← message, stack (getter)
       ↓
   Object.prototype
       ↓
     null

MyError 实例
  __proto__ → MyError.prototype   ← 可能有自定义方法
       ↓
  Error.prototype
       ↓
  Object.prototype
```

### 为什么 `class extends Array` 特殊

ES6 之前 `Array` 不能被 `Object.create` 正确子类化——因为 `Array` 有内部槽位 `[[DefineOwnProperty]]`，`Object.create` 拷贝不了。ES6 的 `class extends Array` 才能正确继承，因为 `extends` 语法会调用 `Array` 的 `[[Construct]]` 内部方法：

```js
// ❌ ES5 方式 —— arr.length 行为异常
function MyArray() { Array.apply(this, arguments); }
MyArray.prototype = Object.create(Array.prototype);
const ma = new MyArray();
ma[0] = 1;
ma.length;  // 0 —— 不会自动更新！内部槽位没正确初始化

// ✅ ES6 class extends —— 正确
class MyArray extends Array {}
const ma = new MyArray();
ma[0] = 1;
ma.length;  // 1 —— 内部槽位正常工作
```

> 这就是为什么第三份简史提到的 "内置方法可以被子类化" 在 ES6 之前不成立。
