# V8 怎么把原型链从 O(n) 优化到 O(1)

> 原型链查找理论上每次都要爬链。V8 用了两层优化让它接近 C++ vtable 的性能。

## 第一层：Hidden Class（隐藏类）

V8 眼里，对象不是"属性的集合"，而是"按照添加顺序形成的结构"。同样顺序、同样名字的属性 → 同一个 Hidden Class。

```js
const a = {};
a.name = '旺财';    // HC1 → HC2 (name 在 offset 0)
a.age = 3;         // HC2 → HC3 (age 在 offset 1)

const b = {};
b.name = '小黑';    // HC1 → HC2 —— 同样的过渡路径
b.age = 5;         // HC2 → HC3 —— 同样的最终 Hidden Class

// 添加顺序变了 = 不同的 Hidden Class
const c = {};
c.age = 3;         // HC1 → HC4 —— 不同路径
c.name = '大黄';   // HC4 → HC5 —— 不同最终 HC
```

> 类比 C++：Hidden Class ≈ C++ 编译期生成的 type layout，区别是 V8 在**运行时**生成。

## 第二层：Inline Cache（内联缓存）

把原型链查找结果缓存在**调用点**，JIT 编译成直接偏移：

```
// V8 在 hot 调用点生成的伪机器码
cmp [rdi + MAP_OFFSET], CACHED_HIDDEN_CLASS  // 对象的 HC 跟上次一样？
jne slow_path                                   // 不一样 → 走慢路径（爬链）
mov rax, [rdi + CACHED_OFFSET]                 // 一样 → 直接从缓存偏移取
```

第一次查原型链，后面 N-1 次直接 `*(obj + 固定偏移)`。前提是调用点保持"单态"。

## 三种状态

| 状态 | HC 种类 | 性能 |
|---|---|---|
| 单态（monomorphic） | 1 种 | 最快——一次 cmp + 直接偏移 |
| 多态（polymorphic） | 2-4 种 | 稍慢——类似 switch 表 |
| 巨态（megamorphic） | > 4 种 | 退化回哈希查找 / 原型链 |

## 什么让 JIT 放弃你

| 操作 | 后果 |
|---|---|
| 构造函数里按不同顺序加属性 | 生成不同 HC，IC 命中率下降 |
| `delete obj.prop` | 打破 HC 链，退化到字典模式（慢路径） |
| 同一函数传 > 4 种不同 shape 的对象 | IC 变巨态 |
| 运行时给原型加属性 | 所有已有对象的 IC 全部失效 |

> Puerts 场景：TS 类在运行时动态增删属性，V8 会退化到慢路径。保持类结构稳定就是帮 V8 作弊。
