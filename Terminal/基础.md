# 终端基础 & iTerm2 常用操作

> 写给从零开始的自己：终端是什么、怎么用、为什么这么用。

---

## 1. 先搞清楚几个概念

刚接触 Mac 最容易混淆的三个东西：

| 你看到的 | 它是什么 | 类比 |
|---------|---------|------|
| **终端（Terminal）** | 一个"窗口程序"，负责显示文字、接收你的输入 | 像 VS Code 的界面壳 |
| **Shell** | 真正执行命令的程序，跑在终端里面 | 像 VS Code 里的编译器 |
| **iTerm2** | 一个第三方的终端程序，比 Mac 自带的「终端.app」强很多 | 像 Chrome vs. Safari |

**所以：iTerm2 是终端的一种，里面跑的通常是 zsh（Mac 默认 shell）。**

> 你可能还听过 bash、fish——这些是不同的 shell，就像 C++、Python、JS 是不同的语言。目前你只需要知道自己的 shell 是 zsh 就够了。

---

## 2. 为什么要用终端？

作为一个写代码的人，终端不是"可选的炫技"，而是：

- **有些事情 GUI 做不了**：比如批量重命名 100 个文件、查看隐藏文件、ssh 远程连服务器
- **效率**：装东西（`brew install xxx`）、跑脚本、git 操作，敲几个字比鼠标点半天快得多
- **理解系统**：Mac 的图形界面底下全是 Unix 命令在跑；学会终端 = 学会 Mac 的"源码层面"

> 你学 Rust/C++ 会碰到 `cargo build`、`cmake`、`gdb`，这些全在终端里跑。

## 3. Shell 命令的心智模型

每一条命令都可以理解成一个句子：

```
命令  [选项]  [参数]
```

比如：

```bash
ls -l /Users/yxqin
# │  │   │
# │  │   └── 参数：对谁操作（目录路径）
# │  └────── 选项：怎么操作（-l 表示"长格式"）
# └───────── 命令：做什么（list，列出文件）
```

**选项（flag）** 通常以 `-` 开头，一个字母的用单横杠 `-l`，多个字母的用双横杠 `--help`。

> 记不住选项很正常，没人能记住。套路是：
> 1. `命令 --help` 看帮助
> 2. `man 命令` 看手册（按 `q` 退出）
> 3. 忘了就搜

## 4. 最短必要命令清单

下面这些是你每天都可能用到的。**不要试图一次背下来**，用到时回来查就行。

### 4.1 我在哪 & 有什么

```bash
pwd          # print working directory — "我现在在哪个目录？"
ls           # list — "这个目录下有什么文件？"
ls -la       # -l 长格式（显示权限、大小、时间） + -a 显示隐藏文件
ls -larth    # 额外：-r 倒序 -t 按时间 -h 人类可读的大小（组合拳，常用）
cd 目录名     # change directory — 去某个目录
cd ..        # 回上一级
cd ~         # 回家目录（/Users/yxqin）
cd -         # 回到刚才在的那个目录
```

### 4.2 文件操作

```bash
mkdir 目录名         # make directory — 创建目录
mkdir -p a/b/c       # 递归创建目录链（parent）
touch 文件名         # 创建空文件，或更新文件的修改时间
rm 文件名            # remove — 删除文件（⚠️ 没有回收站）
rm -r 目录           # 递归删除目录（⚠️⚠️ 非常危险，删之前先 ls 确认）
rm -rf 目录          # 强制递归删除，不提示（⚠️⚠️⚠️ 著名的 rm -rf / 梗）
cp 源 目标           # copy — 复制文件
cp -r 源目录 目标    # 递归复制目录
mv 源 目标           # move — 移动/重命名（移动和重命名是同一个命令！）
```

> ⚠️ **rm 没有回收站**。有个安全习惯：删之前先用 `ls` 或 `echo` 看一下要删的是什么。

### 4.3 查看文件内容

```bash
cat 文件名           # concatenate — 一次性输出整个文件（小文件用）
less 文件名          # 分页查看（大文件用，按 q 退出，/ 搜索）
head -n 20 文件名    # 看前 20 行
tail -n 20 文件名    # 看后 20 行
tail -f 文件名       # follow — 实时追踪文件末尾（看日志专用）
```

### 4.4 搜索

```bash
grep "关键词" 文件名     # 在文件中搜索匹配的行
grep -r "关键词" 目录    # 递归搜索整个目录
grep -i "关键词" 文件    # 忽略大小写
grep -v "关键词" 文件    # 排除匹配的行（反选）

find 目录 -name "*.rs"  # 按文件名查找
find 目录 -type d       # 只找目录
```

> `grep` + `find` 是终端搜索的"双刀"。你后面学 Rust 时可以试试：`grep -r "fn main" _raw/rust-by-example-cn/`。

## 5. 管道和重定向——命令之间的"胶水"

这是终端真正强大的地方：**把简单命令组合成复杂操作。**

### 5.1 重定向：把输出存起来

```bash
# >  覆盖写入文件
echo "hello" > 文件.txt     # 把 "hello" 写进文件（覆盖）

# >> 追加写入文件
echo "world" >> 文件.txt    # 把 "world" 追加到文件末尾
```

### 5.2 管道：把 A 的输出喂给 B

```bash
# |  管道符：左边命令的输出 → 右边命令的输入
ls -la | grep ".rs"         # 列出文件，只显示含 .rs 的行
cat 日志.log | grep "ERROR" | wc -l   # 看日志里有多少行 ERROR
```

**管道的思维模型**：就像流水线上的工序，每个命令做一件事，做好，然后用管道串起来。

> Unix 哲学：每个命令只做一件小事，把它们拼起来就是大能力。

## 6. iTerm2 核心操作

这部分是你和终端交互的"快捷键肌肉记忆"。

### 6.1 标签页（Tab）

| 操作 | 快捷键 | 记忆技巧 |
|------|--------|---------|
| 新建标签 | `Cmd + T` | T = Tab |
| 关闭标签 | `Cmd + W` | W = close Window |
| 切换标签 | `Cmd + ←/→` | 左右切换 |
| 切换标签 | `Cmd + 数字` | 跳到第 N 个 |

### 6.2 分屏（Split Pane）

这是 iTerm2 的杀手级功能——**一个窗口里同时看多个终端**。

| 操作 | 快捷键 | 记忆技巧 |
|------|--------|---------|
| 垂直分屏 | `Cmd + D` | D = Divide vertically |
| 水平分屏 | `Cmd + Shift + D` | Shift = 换个方向 |
| 切换窗格 | `Cmd + Option + 方向键` | Option = 选择方向 |
| 关闭当前窗格 | `Cmd + W` | 同关闭标签 |
| 当前窗格最大化 | `Cmd + Shift + Enter` | 临时的全屏 |

> **什么时候用分屏？** 左边写代码/跑 vim，右边跑测试/看日志；或者上面 ssh 服务器，下面本地操作。

### 6.3 光标操作（比鼠标快得多）

| 操作 | 快捷键 | 记忆技巧 |
|------|--------|---------|
| 跳到行首 | `Ctrl + A` | A = 行首（第一个字母） |
| 跳到行尾 | `Ctrl + E` | E = End |
| 往前删一个词 | `Esc + Backspace` 或 `Ctrl + W` | W = Word |
| 往后删一个词 | `Esc + D`（或配置为 `Option + D`） | D = Delete word |
| 删到行首 | `Ctrl + U` | |
| 删到行尾 | `Ctrl + K` | K = Kill to end |
| 清屏 | `Ctrl + L` 或 `Cmd + K` | L = cLear / K = clear |
| 粘贴 | `Cmd + V` | Mac 通用 |

> 最开始你会习惯用鼠标和方向键——没关系。但每当你觉得"手离开键盘很烦"时，就记一个快捷键。一个月后你就是快捷键老手。

### 6.4 搜索

| 操作 | 快捷键 |
|------|--------|
| 搜索当前屏幕 | `Cmd + F` |
| 全局搜索（所有标签页的文本） | `Cmd + Option + E` |

### 6.5 其他实用功能

- **选中即复制**：iTerm2 里选中文本就自动复制了，不用 `Cmd + C`
- **Cmd + 点击**：点击文件路径、URL 可以直接打开
- **`Cmd + ;`**：自动补全（根据历史命令），比一直按 `↑` 找历史快
- **`Ctrl + R`**：增量搜索历史命令（输入关键词实时匹配），终端最高频操作之一

### 6.6 保存 / 恢复窗口布局

当你调好了一套分屏+标签页的布局（比如左边写代码右边看日志，上面 ssh 服务器下面本地操作），不用每次重新摆：

| 操作 | 快捷键 / 路径 | 说明 |
|------|--------------|------|
| 保存当前布局 | `Cmd + Shift + S` | 起个名字，比如「Rust 开发」 |
| 恢复布局 | `Window` → `Restore Window Arrangement` → 选名称 | 手动恢复 |
| 设为默认布局 | `Window` → `Restore Window Arrangement` → 选名称 → 勾选「Use as default」 | 每次打开 iTerm2 自动恢复 |

> 它会记住：窗口位置和大小、每个标签页、每个分屏的尺寸、每个 pane 当前的工作目录。下次恢复一模一样。

## 7. 日常实用技巧

### 7.1 路径的快捷写法

```bash
.        # 当前目录
..       # 上一级目录
~        # 家目录（/Users/yxqin）
-        # 上一个目录（配合 cd - 用）
```

### 7.2 sudo：借用管理员权限

```bash
sudo 命令    # Super User DO — 以管理员身份执行
```

输入密码时屏幕不显示任何东西（包括 `*`），**这不是卡住了，是安全设计**。盲打密码按回车就好。

### 7.3 中断和退出

```bash
Ctrl + C    # 终止当前正在跑的命令
Ctrl + D    # 发送 EOF（比如退出 python/shell 交互）
q           # 退出 less/man 等分页程序
:q          # 退出 vim（你总会不经意打开它然后不知道怎么退 😂）
```

### 7.4 通配符

```bash
*.rs         # 匹配所有 .rs 文件
*.{rs,toml}  # 匹配 .rs 和 .toml 文件
目录/**/*.rs # ** 递归匹配所有子目录
```

## 8. 第一个练习（建议你真的敲一遍）

打开 iTerm2，依次敲：

```bash
# 1. 看看自己在哪
pwd

# 2. 看看家目录有什么
ls -la ~

# 3. 去桌面建个临时目录玩一下
cd ~/Desktop
mkdir terminal-practice
cd terminal-practice

# 4. 创建文件、写内容、查看
echo "hello terminal" > test.txt
cat test.txt
echo "second line" >> test.txt
cat test.txt

# 5. 试试管道
cat test.txt | grep "hello"

# 6. 玩完删掉
cd ..
rm -r terminal-practice
```

> 如果你敲 `rm -r` 的时候手抖了——恭喜，这是正常反应。保持敬畏。

5. **学 shell 脚本**：当你想把重复操作自动化时，自然就会去写脚本了
